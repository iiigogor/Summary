## 2. Перечень используемых инструментов с обоснованием выбора
### - В качестве языка программирования выбираем Java - проще найти специалистов
### - В качестве тестового фреймворка JUnit(@BeforeEach, @Override ), Docker для запуска автотестов,  фреймворк Selenium WebDriver для автоматизации действий браузера
### - для запросов к API используется REST Assured
### - FAKER или для подготовки тестовых данных ( генерация имени, телефона, эл. почты)
### - Cucumber чтобы уменьшить расходы проекта на написание автотестов
### - Allure для анализа результатов и отчетности

## Сценарии заполнения и отправки формы записи
### *негативные:*
№1

**Шаги воспроизведения**
* Не заполнять поле "Имя".
* Не заполнять поле "Телефон".
* Не заполнять поле "Электронная почта".
* Кликнуть на кнопку "Записаться".

**Ожидаемый результат:**
Запись не произведена. Рядом с полями "Телефон" и "Электронная почта" выводится сообщение "обязательное поле".

№2

**Шаги воспроизведения**
* Заполнить поле "Имя" одним знаком кириллицы . 
* Заполнить поле "Телефон" в формате +7 и 10 цифр.
* Не заполнять поле "Электронная почта"
* Кликнуть на кнопку "Записаться"

**Ожидаемый результат:**
Запись не произведена. Рядом с полями "Имя" выводится сообщение "Не короче 2 символов".Рядом с полем "Электронная почта" выводится сообщение "обязательное поле".

* Кликнуть на кнопку "Записаться"
